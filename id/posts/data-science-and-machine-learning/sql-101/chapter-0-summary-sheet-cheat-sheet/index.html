SQL 101 | Panduan Cepat SQL dan Basis Data

Eugenius Wahyudiarto Jul 22, 2023
sql-101

Panduan cepat ini mencakup mulai dari pernyataan `SELECT` dasar hingga gabungan tingkat lanjut, subkueri, dan pengoptimalan kueri.

Memahami SQL dan Basis Data

  • Konsep Kunci: Tabel, Baris, dan Kolom
    CREATE TABLE karyawan (
    id INT PRIMARY KEY,
    nama VARCHAR(100),
    departemen VARCHAR(50)
);

INSERT INTO karyawan (id, nama, departemen) VALUES (1, 'Alice', 'HR');
SELECT * FROM karyawan;

Menyiapkan Lingkungan Anda

  • Membuat Basis Data
    CREATE DATABASE perusahaan_db;

Mengambil Data dengan Pernyataan SELECT

  • Dasar-Dasar SELECT

    SELECT nama, departemen FROM karyawan;

  • Menyaring Data dengan WHERE Clause

    SELECT * FROM karyawan WHERE departemen = 'HR';

  • Mengurutkan Hasil dengan ORDER BY

    SELECT * FROM karyawan ORDER BY nama ASC;

  • Membatasi Hasil dengan LIMIT

    SELECT * FROM karyawan LIMIT 5;

  • Mengagregasi Data dengan GROUP BY

    SELECT departemen, COUNT(*) FROM karyawan GROUP BY departemen;

  • Menggabungkan Tabel dengan INNER JOIN

    SELECT k.nama, d.nama_departemen 
FROM karyawan k 
INNER JOIN departemen d ON k.departemen = d.id;

Penyaringan Lanjutan

  • Kondisi WHERE yang Kompleks

    SELECT * FROM karyawan WHERE departemen = 'HR' AND nama LIKE 'A%';

  • Pencocokan Pola dengan LIKE

    SELECT * FROM karyawan WHERE nama LIKE '%son';

  • Penyaringan Rentang dengan BETWEEN

    SELECT * FROM karyawan WHERE id BETWEEN 1 AND 10;

  • Pemeriksaan Null

    SELECT * FROM karyawan WHERE departemen IS NULL;

Gabungan Lanjutan dan Subquery

  • LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN

    SELECT k.nama, d.nama_departemen 
FROM karyawan k 
LEFT JOIN departemen d ON k.departemen = d.id;

  • Self-Joins

    SELECT k1.nama, k2.nama AS nama_manager 
FROM karyawan k1 
JOIN karyawan k2 ON k1.manager_id = k2.id;

  • Cross Joins

    SELECT k.nama, p.nama_proyek 
FROM karyawan k 
CROSS JOIN proyek p;

  • Subqueries

    SELECT nama 
FROM karyawan 
WHERE departemen = (SELECT id FROM departemen WHERE nama_departemen = 'IT');

  • Subquery Terkorelasi

    SELECT k1.nama 
FROM karyawan k1 
WHERE k1.gaji > (SELECT AVG(k2.gaji) FROM karyawan k2 WHERE k1.departemen = k2.departemen);

  • Menggunakan EXISTS dan IN dengan Subqueries

    SELECT nama 
FROM karyawan 
WHERE EXISTS (SELECT 1 FROM departemen WHERE nama_departemen = 'IT');

Operasi Set dan Common Table Expressions (CTEs)

  • UNION, UNION ALL

    SELECT nama FROM karyawan WHERE departemen = 'HR'
UNION
SELECT nama FROM karyawan WHERE departemen = 'IT';

  • Menggunakan WITH untuk Mendefinisikan CTEs

    WITH hitung_departemen AS (
    SELECT departemen, COUNT(*) AS jumlah_karyawan 
    FROM karyawan 
    GROUP BY departemen
)
SELECT * FROM hitung_departemen WHERE jumlah_karyawan > 10;

Fungsi Window dan Agregat Lanjutan

  • Menggunakan OVER dan PARTITION BY

    SELECT nama, gaji, RANK() OVER (PARTITION BY departemen ORDER BY gaji DESC) as peringkat_departemen
FROM karyawan;

  • Fungsi Peringkat

    SELECT nama, DENSE_RANK() OVER (ORDER BY gaji DESC) as peringkat_gaji
FROM karyawan;

  • HAVING Clause untuk Menyaring Kelompok

    SELECT departemen, COUNT(*) 
FROM karyawan 
GROUP BY departemen 
HAVING COUNT(*) > 5;

Pencarian Teks Penuh dan Manipulasi Data

  • Query Pencarian Teks Penuh

    SELECT * FROM dokumen WHERE MATCH(teks) AGAINST('kueri pencarian');

  • Fungsi Tanggal dan Waktu

    SELECT nama, DATE_FORMAT(tanggal_masuk, '%Y-%m-%d') FROM karyawan;

Bekerja dengan Views

  • Membuat dan Mengelola Views
    CREATE VIEW karyawan_hr AS 
SELECT nama, departemen 
FROM karyawan 
WHERE departemen = 'HR';

Optimasi Query Lanjutan

  • Memahami Rencana Eksekusi Query

    EXPLAIN SELECT * FROM karyawan WHERE departemen = 'HR';

  • Menggunakan Indeks untuk Optimasi Performa

    CREATE INDEX idx_departemen ON karyawan(departemen);
