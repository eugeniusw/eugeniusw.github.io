SQL 101 | Bab 2: Menyiapkan Lingkungan Basis Data Anda

Eugenius Wahyudiarto Jan 20, 2024
sql-101

Menyiapkan lingkungan basis data yang kuat dan efisien merupakan langkah penting bagi bisnis dan individu. Baik Anda mengelola data untuk proyek kecil atau aplikasi perusahaan berskala besar, menjaga lingkungan basis data Anda dengan benar sangatlah penting untuk kelancaran operasional. Dalam panduan ini, kita akan memandu Anda melalui langkah-langkah penting dalam menyiapkan lingkungan basis data Anda, termasuk memilih Sistem Manajemen Basis Data / Database Management System (DBMS) yang tepat, menginstalnya bersama dengan klien SQL, dan membuat basis data pertama Anda.

Memilih Sistem Manajemen Basis Data / Database Management System (DBMS)

Sebelum mendalami aspek teknis dalam menyiapkan lingkungan basis data, Anda perlu membuat keputusan mendasar: memilih Sistem Manajemen Basis Data / Database Management System (DBMS) yang tepat. DBMS adalah perangkat lunak yang akan mengelola data Anda, dan pilihan yang Anda buat di sini akan berdampak signifikan terhadap keberhasilan proyek Anda. Untuk menentukan pilihan, Anda perlu mempertimbangkan faktor-faktor seperti skalabilitas, kompatibilitas, dan fitur.

  • Skalabilitas: Mengacu pada kemampuan sistem untuk menangani peningkatan beban kerja atau peningkatan volume data tanpa mengorbankan performa atau ketersediaan. Skalabilitas adalah pertimbangan penting karena aplikasi Anda mungkin dimulai dari kecil namun dapat berkembang secara signifikan seiring berjalannya waktu.
  • Kompatibilitas: Ini melibatkan kemampuan DBMS untuk bekerja secara lancar dengan tumpukan perangkat lunak yang ada, termasuk sistem operasi, bahasa pemrograman, dan alat pihak ketiga. Ketidakcocokan dapat menyebabkan masalah integrasi, peningkatan waktu pengembangan, dan tantangan pemeliharaan.
  • Fitur: Fitur-fitur yang ditawarkan oleh DBMS memainkan peran penting dalam menentukan kemampuan dan kesesuaiannya untuk kasus penggunaan tertentu. Fitur dapat sangat bervariasi antara sistem basis data yang berbeda.

Setelah memperhitungkan faktor-faktor tersebut, Anda akan menemukan bahwa ada beberapa pilihan DBMS yang tersedia, masing-masing dengan kekuatan dan kelemahannya sendiri. Berikut beberapa pilihan populer:

  • MySQL: Sistem manajemen basis data relasional sumber terbuka yang terkenal dengan kinerja, keandalan, dan kemudahan penggunaannya. Ini adalah pilihan tepat untuk aplikasi web dan proyek skala kecil hingga menengah.
  • PostgreSQL: DBMS relasional sumber terbuka lainnya yang unggul dalam menangani kueri kompleks dan kumpulan data besar. Ini adalah pilihan tepat untuk aplikasi yang memerlukan skalabilitas dan integritas data.
  • MongoDB: Basis data NoSQL yang ideal untuk menangani data tidak terstruktur atau semi-terstruktur. MongoDB sangat cocok untuk proyek dengan skema data yang berkembang pesat.
  • Microsoft SQL Server: DBMS tangguh dengan dukungan kuat untuk aplikasi berbasis Windows. Ini cocok untuk bisnis yang banyak berinvestasi di ekosistem Microsoft.
  • Oracle Database: DBMS canggih yang dirancang untuk perusahaan besar dengan volume data tinggi dan persyaratan kompleks. Ia menawarkan fitur-fitur canggih untuk manajemen dan keamanan data.

Menginstal DBMS dan Klien SQL

Setelah Anda memilih DBMS, langkah berikutnya adalah menginstalnya pada sistem Anda. Proses instalasi mungkin berbeda-beda tergantung pada sistem operasi Anda dan DBMS spesifik yang Anda pilih. Berikut adalah garis besar umum langkah-langkah yang terlibat:

  • Unduh DBMS: Kunjungi situs web resmi DBMS pilihan Anda dan unduh paket instalasi yang sesuai untuk sistem operasi Anda. Kebanyakan penyedia DBMS menawarkan instruksi yang jelas dan link download di situs web mereka.
  • Instal DBMS: Jalankan paket instalasi dan ikuti petunjuk di layar. Biasanya Anda perlu mengonfigurasi pengaturan seperti direktori instalasi, nomor port, dan kata sandi administratif pada langkah ini.
  • Instal Klien SQL: Selain DBMS, Anda memerlukan klien SQL untuk berinteraksi dengan basis data. Beberapa DBMS dilengkapi dengan klien grafisnya sendiri, namun Anda juga dapat memilih dari berbagai opsi pihak ketiga. Klien SQL yang populer termasuk SQL Server Management Studio, DBeaver, dan MySQL Workbench.
  • Konfigurasi Klien SQL: Setelah klien SQL diinstal, Anda harus mengonfigurasinya agar terhubung ke DBMS yang baru diinstal. Anda harus memberikan detail koneksi seperti alamat server, nomor port, dan kredensial autentikasi.
  • Uji Koneksi: Setelah mengonfigurasi klien SQL Anda, uji koneksi untuk memastikan bahwa koneksi dapat terhubung ke DBMS tanpa masalah apa pun. Jika semuanya sudah diatur dengan benar, Anda siap melanjutkan ke langkah berikutnya.

Dalam panduan SQL 101 ini, PostgreSQL akan digunakan sebagai DBMS karena rangkaian fiturnya yang luas, kemampuan skalabilitas yang luar biasa, dan dukungan yang sangat berharga dari komunitas berdedikasi, menjadikannya pilihan yang sangat baik bagi mereka yang baru mengenal SQL. Sepanjang panduan ini, saya akan mencoba untuk mematuhi standar ANSI SQL, memastikan bahwa pengetahuan yang Anda peroleh dapat diterapkan dengan lancar di berbagai platform DBMS.

Untuk metode instalasi, mari kita sederhanakan prosesnya menggunakan Docker. Ikuti langkah-langkah di bawah ini:

Langkah 1: Instal Docker
Mulailah dengan menginstal Docker, sebuah platform untuk containerisasi yang mengelola pengaturan DBMS kita. Unduh dan instal buruh pelabuhan di sini: https://docs.docker.com/get-docker/ Validasi instalasi Docker Anda dengan menjalankan docker -v di baris perintah/terminal.

Langkah 2: Buat File docker-compose.yml
Selanjutnya, buat file docker-compose.yml.

version: '3'
volumes:
  db_data:
  pgadmin-data:
  docker-entrypoint:

services:
  postgres:
    image: postgres:13.11-bullseye # silahkan diperbarui
    restart: always
    environment:
      POSTGRES_USER: admin # silahkan diubah
      POSTGRES_PASSWORD: admin # silahkan diubah
      POSTGRES_DB: sql101_by_ardenov # silahkan diubah
    volumes:
      - docker-entrypoint:/docker-entrypoint-initdb.d
      - db_data:/var/lib/postgresql/data
  
  pgadmin:
    image: dpage/pgadmin4:7.7 # silahkan diperbarui
    depends_on:
      - postgres
    ports:
      - "8001:80"
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com # silahkan diubah
      PGADMIN_DEFAULT_PASSWORD: admin # silahkan diubah
    volumes:
     - pgadmin-data:/var/lib/pgadmin
    restart: unless-stopped

Langkah 3: Instal DBMS

docker-compose up -d

Langkah 4: Validasi Instalasi DBMS
Kunjungi Klien SQL dengan mengetik localhost:8001 di browser Anda atau dengan mengklik di desktop Docker.

Anda seharusnya menemukan halaman login pgAdmin. Masukkan kredensial login dari file YAML.

Membuat Basis Data

Dengan DBMS dan klien SQL Anda, sekarang saatnya membuat basis data pertama Anda. Membuat basis data melibatkan pendefinisian struktur dan skemanya, yang akan menentukan bagaimana data diatur dan disimpan. Berikut panduan sederhana untuk membuat basis data:

Langkah 1: Luncurkan Klien SQL Anda

Buka klien SQL Anda dan sambungkan ke DBMS menggunakan kredensial yang Anda berikan selama instalasi.

Langkah 2: Buat Basis Data Baru

Di alat kueri / query tool, Anda bisa membuat basis data baru dengan menjalankan perintah SQL seperti:

CREATE DATABASE NamaBasisDataAnda;

Ganti NamaBasisDataAnda dengan nama yang ingin Anda berikan pada basis data Anda.

Langkah 3: Tentukan Tabel & Sisipkan Data

Setelah basis data Anda dibuat, Anda dapat menentukan tabel dan menyimpan data Anda yang akan kita bahas detailnya di bab berikutnya. Setiap tabel harus memiliki struktur yang ditentukan, termasuk kolom dan tipe data.

Langkah 4: Kueri basis data Anda

Anda dapat menjalankan pernyataan SQL SELECT untuk mengambil dan memanipulasi data dalam basis data Anda. Untuk saat ini kita menggunakannya untuk memeriksa apakah pengaturan Anda sudah selesai.

SELECT 'Hello World from SQL' AS output

Jika Anda melihat hasilnya, selamat! Anda telah berhasil menyiapkan lingkungan basis data Anda, termasuk memilih DBMS yang tepat, menginstalnya bersama dengan klien SQL, dan membuat basis data pertama Anda.

Kesimpulan

Menyiapkan lingkungan basis data Anda adalah langkah pertama yang penting dalam setiap proyek berbasis data. Dengan hati-hati memilih DBMS yang tepat, menginstalnya dengan benar, dan membuat basis data yang terstruktur dengan baik, Anda meletakkan dasar untuk manajemen data dan pengembangan aplikasi yang efisien. Di bab selanjutnya, kita akan mengambil data dengan pernyataan SELECT. Sampai jumpa lagi, selamat berkueri!
