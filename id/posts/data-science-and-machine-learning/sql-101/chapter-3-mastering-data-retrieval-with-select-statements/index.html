SQL 101 | Bab 3: Menguasai Pengambilan Data dengan Pernyataan SELECT

Eugenius Wahyudiarto Jul 20, 2024
sql-101

Pelajari cara menguasai pengambilan data dengan SQL dalam panduan komprehensif ini. Jelajahi dasar-dasar SELECT, memfilter dengan WHERE, mengurutkan dengan ORDER BY, membatasi hasil dengan LIMIT, menggabungkan data dengan GROUP BY, dan menggabungkan tabel dengan JOIN. Pendahuluan

Bahasa Pertanyaan Terstruktur / Structured Query Language (SQL) adalah tulang punggung pengelolaan dan manipulasi data dalam basis data relasional. Salah satu operasinya yang paling mendasar adalah mengambil data, dan pernyataan SELECT adalah pemain kunci dalam permainan ini. Dalam panduan ini, kita akan mempelajari esensi pengambilan data, menjelajahi dasar-dasar SELECT, menyaring dengan WHERE, mengurutkan dengan ORDER BY, membatasi hasil dengan LIMIT, menggabungkan data dengan GROUP BY, dan terakhir, kekuatan menggabungkan tabel dengan JOIN.

Persiapan

Sebelum memulai, kita perlu membuat tabel untuk melatih pernyataan SQL SELECT. Kita akan menggunakan pgAdmin sebagai tempat berlatih SQL kita. Silakan merujuk ke bab sebelumnya jika Anda belum menginstalnya. Ada tiga tabel yang akan kita gunakan: books, authors, dan sales. Untuk saat ini, jangan khawatir tentang sintaks cara menyiapkan tabel-tabel ini, karena akan dijelaskan di bab berikutnya.

Jika Anda lebih suka mengisi tabel menggunakan file CSV, Anda dapat mengunduhnya di sini: books.csv, authors.csv, sales.csv.

Membuat dan mengisi tabel books:

-- membuat tabel "books"
CREATE TABLE books (
  book_id SERIAL PRIMARY KEY,
  book_title VARCHAR(255) NOT NULL,
  book_genre VARCHAR(255),
  book_summary TEXT,
  total_pages INT,
  author_id INT
);

-- mengisi 100 data sampel ke dalam tabel "books"
INSERT INTO books (book_title, book_genre, book_summary, total_pages, author_id)
SELECT
  CONCAT('Book ', seq) AS book_title,
  CASE seq % 3
    WHEN 0 THEN 'Fiction'
    WHEN 1 THEN 'Non-Fiction'
    ELSE 'Biography'
  END AS book_genre,
  CONCAT('This is a summary of Book ', seq) AS book_summary,
  seq * 100 AS total_pages,
  (seq % 10) + 2 AS author_id
FROM
  generate_series(1, 100) AS seq;

Membuat dan mengisi tabel sales:

-- membuat tabel "sales"
CREATE TABLE sales (
    sale_id SERIAL PRIMARY KEY,
    book_id INTEGER,
    timestamp_sale TIMESTAMPTZ,
    date_sale DATE,
    price DECIMAL(10, 2),
    cost DECIMAL(10, 2)
);

-- mengisi 100 data sampel ke dalam tabel "sales"
SELECT SETSEED(0.5);

INSERT INTO sales (book_id, timestamp_sale, date_sale, price, cost)
SELECT
    (FLOOR(RANDOM() * 100) + 1) AS book_id,
    (TIMESTAMP '2022-08-08 13:10:11' - ((RANDOM() * (365 * 24 * 60 * 60))::integer || ' seconds')::interval) AT TIME ZONE 'UTC' AS timestamp_sale,
    DATE '2022-08-08' - (RANDOM() * 365)::integer AS date_sale,
    ROUND((RANDOM() * 100 + 10)::numeric, 2)+100 AS price,
    ROUND((RANDOM() * 50 + 5)::numeric, 2) AS cost
FROM
    generate_series(1, 100);

-- sinkronisasi date yang dihasilkan berdasarkan timestamp yang dihasilkan
UPDATE sales
SET date_sale = (
  SELECT date(timestamp_sale)
  FROM sales AS t2
  WHERE t2.sale_id = sales.sale_id
);

Dasar SELECT

Pada intinya, pernyataan SELECT adalah tentang mengambil data dari tabel tertentu. Sintaksnya mudah:

SELECT kolom_1, kolom_2
FROM tabel_1;

Pernyataan ini memilih kolom tertentu dari sebuah tabel, memberikan tampilan data yang terstruktur.

Contoh: Kueri SQL ini mengambil kolom book_id, book_title, dan total_pages dari tabel books.

SELECT book_id, book_title, total_pages
FROM books;

Di bawah ini adalah hasil yang mungkin Anda lihat saat menjalankan kueri ini:

Menyaring data Menggunakan Klausa WHERE

Untuk menyempurnakan hasil Anda, klausa WHERE ikut berperan. Ini memungkinkan Anda menyaring baris berdasarkan kondisi tertentu:

SELECT kolom_1, kolom_2
FROM tabel_1
WHERE kondisi;

Contoh: Kueri SQL ini mengambil kolom book_id, book_title, dan total_pages dari tabel books namun hanya untuk buku yang total_pages-nya lebih besar dari 500.

SELECT book_id, book_title, total_pages
FROM books
WHERE total_pages > 500;

Berikut ini pratinjau hasil yang terlihat saat Anda menjalankan kueri ini:

Mengurutkan Hasil Menggunakan Klausa ORDER BY

ORDER BY memungkinkan Anda mengurutkan kumpulan hasil berdasarkan satu atau beberapa kolom, baik dalam urutan menaik (ASC) atau menurun (DESC):

SELECT kolom_1, kolom_2
FROM tabel_1
ORDER BY kolom_1 ASC/DESC;

Contoh: Kueri SQL ini mengambil kolom book_id, book_title, dan total_pages dari tabel books dan mengurutkan hasilnya berdasarkan total_pages dalam urutan menurun, sehingga buku dengan halaman terbanyak muncul Pertama.

SELECT book_id, book_title, total_pages
FROM books
ORDER BY total_pages DESC;

Berikut cuplikan hasil yang Anda peroleh dari menjalankan kueri ini:

Membatasi Hasil Menggunakan Klausa LIMIT

Saat menangani kumpulan data besar, LIMIT sangat penting. Ini membatasi jumlah baris yang dihasilkan oleh kueri:

SELECT kolom_1, kolom_2
FROM tabel_1
LIMIT jumlah_baris;

Contoh: Kueri SQL ini mengambil kolom book_id, book_title, dan total_pages dari tabel books, namun membatasi hasil hanya pada 8 data pertama.

SELECT book_id, book_title, total_pages
FROM books
LIMIT 8;

Berikut sekilas data yang Anda lihat saat menjalankan kueri ini:

Menggabungkan Data Menggunakan Klausa GROUP BY

GROUP BY digunakan untuk menggabungkan data berdasarkan kolom tertentu. Ini sering dipasangkan dengan fungsi agregat seperti COUNT, SUM, AVG, dll.:

SELECT kolom_1, fungsi_aggregat(kolom_2)
FROM tabel_1
GROUP BY kolom_1;

Contoh: Kueri SQL ini menghitung jumlah buku untuk setiap genre dengan mengelompokkan hasilnya berdasarkan book_genre.

SELECT
  book_genre,
  COUNT(book_id)
FROM books
GROUP BY book_genre;

Menggabungkan Tabel Menggunakan Klausa JOIN atau INNER JOIN

Dalam basis data relasional, data sering kali didistribusikan ke beberapa tabel. JOIN atau INNER JOIN membantu menggabungkan baris dari dua tabel atau lebih berdasarkan kolom terkait:

SELECT kolom_1, kolom_2
FROM tabel_1
INNER JOIN tabel_2 ON tabel_1.kolom_x = tabel_2.kolom_x;

Contoh: Kueri SQL ini mengambil book_id, book_title, dan author_name dengan menggabungkan tabel books dengan tabel authors berdasarkan pencocokan author_id.

SELECT
  books.book_id,
  books.book_title,
  authors.author_name
FROM books
INNER JOIN authors
  ON books.author_id = authors.author_id;

Jenis-jenis penggabungan

FULL JOIN
Menampilkan semua baris dari kedua tabel, baris yang cocok dan baris yang tidak cocok. Jika suatu baris tidak memiliki kecocokan di tabel lainnya, maka baris tersebut akan tetap disertakan dalam hasil.

INNER JOIN atau JOIN
Hanya menampilkan baris yang cocok dari kedua tabel. Jika suatu baris tidak memiliki kecocokan di tabel lainnya, maka baris tersebut akan dikeluarkan dari hasil.

LEFT JOIN
Menampilkan semua baris dari tabel kiri (pertama) dan baris-baris yang cocok dari tabel kanan (kedua). Jika suatu baris tidak memiliki kecocokan di tabel kanan, maka baris tersebut akan tetap disertakan dalam hasil.

RIGHT JOIN
Menampilkan semua baris dari tabel kanan (kedua) dan baris-baris yang cocok dari tabel kiri (pertama). Jika suatu baris tidak memiliki kecocokan di tabel sebelah kiri, maka baris tersebut akan tetap dimasukkan dalam hasil.

CROSS JOIN atau ,
Menampilkan produk Kartesius dari kedua tabel. Menghasilkan seluruh kemungkinan kombinasi baris dari kedua tabel.

UNION ALL
Menggabungkan hasil dari dua atau lebih pernyataan SELECT ke dalam satu rangkaian hasil.

UNION
Menggabungkan hasil dari dua atau lebih pernyataan SELECT ke dalam satu kumpulan hasil dan menghapus baris duplikat dari kumpulan hasil.

Memeriksa LEFT JOIN pada Hubungan Satu-ke-Banyak

Dalam kueri SQL ini, kita memeriksa bagaimana LEFT JOIN beroperasi dalam hubungan satu-ke-banyak antara dua tabel: authors dan books. Dalam skenario ini, satu penulis dapat dikaitkan dengan beberapa buku.

LEFT JOIN memastikan bahwa semua data dari tabel authors disertakan dalam kumpulan hasil, terlepas dari apakah ada data yang cocok dalam tabel books. Jika seorang penulis tidak mempunyai buku, informasinya akan tetap muncul di hasil, dengan nilai NULL di kolom book_title.

SELECT
  authors.author_id,
  authors.author_name,
  books.book_title
FROM authors
LEFT JOIN books
  USING (author_id)
ORDER BY author_id;

Kuis

Sekarang, cobalah menjawab kuis di bawah ini untuk memperdalam pemahaman Anda.

Sebutkan 3 buku teratas yang:

  • Ditulis oleh penulis paling populer
  • Memiliki lebih dari 500 halaman, dengan preferensi yang mendekati 500

Sertakan kolom berikut dari daftar: book_id, book_title, total_pages, author_name, popularity

Klik untuk mengungkap jawabannya

[!TIP] Coba jalankan dan bandingkan dengan jawaban Anda.

SELECT
  book_id,
  book_title,
  total_pages,
  author_name,
  popularity
FROM authors
LEFT JOIN books
  USING(author_id)
WHERE total_pages > 500
ORDER BY popularity DESC, total_pages ASC
LIMIT 3

Kesimpulan

Menguasai pengambilan data dalam SQL membuka banyak kemungkinan untuk mengekstraksi wawasan dari basis data secara efisien. Dengan memahami nuansa dasar-dasar SELECT, menyaring dengan WHERE, mengurutkan dengan ORDER BY, membatasi dengan LIMIT, menggabungkan dengan GROUP BY, dan menggabungkan tabel dengan JOIN, Anda memberdayakan diri Anda untuk mengatasi beragam skenario data.

Dalam panduan ini, kita baru saja mulai mengeksplorasi kemungkinannya. SQL adalah bahasa yang kuat, dan semakin banyak Anda berlatih, semakin besar Anda akan menemukan potensinya. Saat Anda menjadi lebih terampil dalam menulis kueri SQL, Anda akan lebih mudah mengubah data mentah menjadi wawasan yang berharga. Jadi, bersiaplah, buka konsol SQL Anda, dan selami dunia pengambilan data.
