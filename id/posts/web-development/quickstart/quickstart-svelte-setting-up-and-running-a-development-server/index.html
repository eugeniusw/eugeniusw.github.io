Mulai Cepat Svelte - Menyiapkan dan Menjalankan Server Pengembangan

Eugenius Wahyudiarto Nov 28, 2024
web-dev

Mulai cepat proyek Svelte dan menjalankan server pengembangannya menggunakan Docker. Panduan ini menjelaskan cara mengonfigurasi docker-compose.yml untuk menginisialisasi proyek Svelte + Vite dan menjalankan server pengembangan dengan hot-reloading.

Dalam panduan mulai cepat ini, Svelte akan diinstal menggunakan Docker. Pendekatan ini berarti Anda tidak perlu menginstal Node.js atau dependensi apa pun secara langsung di komputer Anda, karena semuanya beroperasi dalam lingkungan kontainer yang bersih.

Menginstall Svelte

Buatlah file docker-compose.yml yang isinya sebagai berikut:

version: '3'
services:
  npm:
    image: node:18-alpine # LTS
    volumes:
      - ./app:/app:delegated
      - ./.npm:/root/.npm:delegated
    ports:
      - "5500:5173"
    command:
      - /bin/sh
      - -c
      - |
        npm create vite@5.5.5 . -- --template svelte
        npm install        
    working_dir: /app
    restart: no

File docker-compose.yml ini menyiapkan lingkungan yang dikontainerisasi untuk membuat proyek Svelte menggunakan Node.js. File ini menggunakan citra node:18-alpine yang ringan (~50 MB) untuk memastikan lingkungan pengembangan yang konsisten, sehingga tidak perlu untuk menginstal Node.js di komputer lokal Anda.

File proyek dibuat dalam folder bersama yang disebut app, sehingga dapat diakses dari sistem host Anda. Konfigurasi menjalankan perintah npm create vite@5.5.5 . -- --template svelte untuk menginisialisasi proyek Svelte yang baru dan menginstal dependensi yang diperlukan menggunakan npm install.

Titik (.) menunjukkan bahwa proyek baru harus dibuat di direktori kerja saat ini. Alih-alih membuat subfolder baru untuk proyek tersebut, file-file tersebut akan ditempatkan langsung di lokasi tempat perintah dijalankan. Pendekatan ini berguna jika Anda telah menyiapkan direktori untuk proyek tersebut, seperti dalam pengaturan Docker tempat /app berfungsi sebagai direktori kerja.

Selain itu, kontainer memetakan port bawaan dari 5173 ke 5500 sehingga Anda dapat mengakses server pengembangan di http://localhost:5500.

Setelah kontainer selesai disiapkan, Anda akan menemukan berkas proyek Svelte di direktori ./app. Dengan menjalankan docker compose up, Anda akan memiliki lingkungan pengembangan Svelte yang berfungsi penuh dan terisolasi dari sistem Anda. Pengaturan ini ideal untuk memastikan kompatibilitas dan menghindari konflik ketergantungan di lingkungan pengembangan lokal Anda.

Jalankan perintah berikut untuk menyiapkan Svelte:

docker compose up

Jalankan Server Pengembangan Svelte

Ganti konten file docker-compose.yml Anda dengan konfigurasi yang diperbarui berikut:

version: '3'
services:
  npm:
    image: node:18-alpine # LTS
    volumes:
      - ./app:/app:delegated
      - ./.npm:/root/.npm:delegated
    ports:
      - "5500:5173"
    command:
      - /bin/sh
      - -c
      - |
        npm run dev -- --host 0.0.0.0        
    working_dir: /app
    restart: no

Perintah tersebut sekarang menjalankan npm run dev, yang memulai server pengembangan Svelte. Tanda -- --host 0.0.0.0 memastikan server terikat ke semua antarmuka jaringan, sehingga dapat diakses dari luar kontainer. Hal ini penting karena kontainer Docker memiliki jaringan terisolasi sendiri, dan pengikatan ke 0.0.0.0 memungkinkan komputer host Anda mengakses server.

Sekarang, jalankan kembali perintah berikut untuk memulai server pengembangan:

docker compose up

Anda dapat mengaksesnya dengan browser di http://localhost:5500 atau di LAN Anda menggunakan alamat ip lokal host (perangkat/komputer): http://[ip_lokal_host]:5500. Saat Anda mengubah file di direktori ./app, browser akan memperbarui secara real time (langsung) berkat kemampuan hot-reload Vite.
